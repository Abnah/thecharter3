Goal → Mission → Vision Flow

🔄 8. Methodology & Approach

▼

Hybrid–Iterative Approach

Why Hybrid–Iterative Works: My requirements are mostly clear but still evolve with feedback. The semester has fixed milestones (predictive backbone), while each major deliverable – resume, portfolio, interviews, and presentation – will go through feedback loops (iterative refinement). This balance keeps me on schedule and allows continuous improvement based on professor, Career Center, and peer input.

🛠️ 9. Project Tool Selection

Primary: Microsoft Project • Click for justification
▼

Selected Tool: Microsoft Project

Why Microsoft Project?

Microsoft Project provides comprehensive project management capabilities that align perfectly with the hybrid-iterative methodology and fixed semester timeline. Its robust scheduling, resource management, and tracking features support both the predictive backbone and iterative refinement cycles.

Key Benefits for This Project

Gantt Chart Visualization: Provides clear visual timeline of all deliverables across the 16-week semester, making it easy to identify dependencies and critical path items (like the portfolio).

Dependency Management: Explicitly tracks how deliverables depend on each other (e.g., portfolio depends on resume completion), ensuring proper sequencing of work.

Resource Allocation: Tracks the 63 hours of self-labor across tasks, helping balance workload and prevent overcommitment during busy weeks.

Baseline Tracking: Allows me to set an initial plan and compare actual progress against it, supporting the iterative feedback loops while maintaining semester milestones.

Milestone Management: Clear marking of key deadlines (Career Center reviews, networking events, application submissions, final presentation) ensures nothing falls through the cracks.

Professional Reporting: Generates professional status reports for professor updates, showing progress, upcoming tasks, and any schedule variances.

Project Structure:

  • Phases: Planning (Weeks 1-5), Execution (Weeks 6-14), Delivery (Weeks 15-16)
  • Work Breakdown Structure: 8 major deliverables broken into subtasks with estimates
  • Milestones: Resume complete, Portfolio live, First networking event, Applications submitted, Presentation ready
  • Tracking: Weekly updates to track % complete, actual hours vs. estimates, and identify schedule slippage early

Integration with Methodology: Microsoft Project's ability to handle both fixed milestones (predictive) and task-level updates (iterative) makes it ideal for the hybrid approach. I can maintain the overall semester schedule while adjusting individual task details based on feedback without disrupting the critical path.

Alternatives Considered: Trello and Asana were evaluated but lack the sophisticated scheduling and dependency management needed for tracking 8 interconnected deliverables across a fixed timeline. Microsoft Project provides the rigor appropriate for demonstrating professional project management skills.
