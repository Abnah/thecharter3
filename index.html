🎯1. Project Title & Description ▼

Launching My IT Career Readiness

This project positions me to secure at least one real interview in IT support/operations by semester-end through targeted application materials, strategic networking, and active job applications. The professional portfolio serves as a key deliverable to support this goal, not as the goal itself.

By combining polished career documents, hands-on networking experiences, and structured interview preparation, this project creates a comprehensive pathway from academic preparation to professional opportunity in the IT field.

⚙️8. Methodology Diagram & Justification ▼

Hybrid Approach: Predictive + Adaptive

Rationale: This project requires a hybrid methodology because it combines fixed semester deadlines and sequential dependencies (predictive elements) with content that benefits from iterative feedback and continuous improvement (adaptive elements). The structure provides clarity while maintaining flexibility for quality refinement.

Detailed Methodology Justification

Why Hybrid Works Best:

  • Predictive Elements: Clear semester timeline with fixed milestones (charter due, presentation date), sequential dependencies (resume must precede portfolio), and defined success criteria
  • Adaptive Elements: Content quality improves through feedback cycles, design iterations respond to stakeholder input, and learning occurs progressively throughout execution
  • Integration: The waterfall phases provide structure and accountability, while sprints within Execute enable continuous improvement without missing deadlines

Sprint Details:

  • Sprint 1 (Weeks 1-2): Draft and refine resume; begin career plan; gather initial feedback
  • Sprint 2 (Weeks 3-4): Build portfolio website; conduct professional interviews; synthesize insights
  • Sprint 3 (Weeks 5-6): Attend networking events; complete mock interviews; apply feedback
  • Sprint 4 (Week 7): Finalize all deliverables; create presentation; prepare for delivery

Each sprint includes: Planning (define sprint goals) → Execution (create/revise deliverables) → Review (gather feedback) → Retrospective (document lessons learned) → Adapt (apply improvements in next sprint)

🛠️9. Project Tool Selection & Justification ▼

Primary Tool: Microsoft Project

Selected as the primary project management tool for this hybrid approach.

Why Microsoft Project is the Best Fit:

  • Hybrid Methodology Support: Provides robust Gantt charts and dependency tracking for the predictive backbone, while allowing task updates and baseline comparisons for iterative sprint management
  • Timeline Management: Clear visualization of fixed semester milestones, critical path identification, and deadline tracking ensure on-time delivery
  • Resource Planning: Tracks 63 hours of labor allocation across deliverables, identifies time conflicts, and supports workload balancing
  • Dependency Tracking: Models sequential relationships (e.g., resume must be complete before portfolio integration) using Finish-to-Start (FS) and Start-to-Start (SS) dependencies
  • Progress Monitoring: Percentage complete tracking, variance analysis, and earned value management provide clear visibility into project health

Key Features Utilized:

  • Work Breakdown Structure (WBS): Hierarchical decomposition of deliverables into manageable tasks
  • Milestones: Key checkpoints like "Resume Finalized," "Portfolio Live," "Interview #1 Complete"
  • Baselines: Saved initial plan for comparing actual vs. planned progress
  • Resource Allocation: Time estimates per task enable realistic scheduling
  • Critical Path: Identifies must-complete-on-time tasks to achieve the interview goal
  • Reports & Dashboards: Visual progress summaries for stakeholder communication

Alternatives Considered:
